## Introduction

## Core Concepts

### Component Architecture

All components follow these patterns:

1. **Functional Components**: No class components
2. **TypeScript**: Strict typing for all props and state
3. **Composition**: Small, reusable components
4. **Accessibility**: WCAG AA+ compliance via Radix UI
5. **Performance**: Lazy loading and code splitting

### State Management

- **Local State**: useState for component-specific state
- **Context API**: AuthContext and CompanyContext for global state
- **Form State**: React Hook Form for complex forms
- **Server State**: React Query (coming soon) for API data

### Styling Approach

- **TailwindCSS**: Utility-first CSS for rapid development
- **CSS Variables**: Theme colors defined as CSS custom properties
- **OKLCH Color Space**: Better perceptual uniformity than RGB/HSL
- **Responsive**: Mobile-first with breakpoints (sm, md, lg, xl, 2xl)
- **Dark Mode**: Automatic with `next-themes` and `.dark` class

## Development Workflow

### Local Development

```bash
# Start development server
cd zarna-frontend
npm run dev

# Server starts at http://localhost:3000
# Hot module replacement enabled
```

### Key Commands

```bash
npm run dev          # Start development server
npm run build        # Build for production (tsc + vite build)
npm run preview      # Preview production build locally
npm run lint         # Run ESLint
npm run format       # Format code with Prettier
npm run format:check # Check code formatting
```

### Path Aliases

Import using `@/` prefix for clean imports:

```typescript
// ✅ Good
import { Button } from "@/components/ui/button"
import { cn } from "@/lib/utils"
import { useAuth } from "@/hooks/use-auth"

// ❌ Bad
import { Button } from "../../../components/ui/button"
```

**Configuration**: `tsconfig.json` paths setting

## Best Practices

The codebase follows strict best practices for maintainability and scalability:

1. **Always use `cn()` for className merging** - Intelligently merges Tailwind classes
2. **Use CSS variables for colors** - Theme-aware, works with dark mode
3. **Type everything** - No `any` types, strict TypeScript
4. **Extract reusable logic to hooks** - DRY principle
5. **Composable components** - Small, focused, reusable
6. **Accessibility first** - Semantic HTML, ARIA labels, keyboard navigation

## Adding Components

### Using shadcn/ui Components

```bash
# Add a new component
npx shadcn@latest add [component-name]

# Examples:
npx shadcn@latest add button
npx shadcn@latest add dialog
npx shadcn@latest add form
```

<Warning>
  **Never manually edit files in `components/ui/`!** Always use the shadcn CLI to add or update components.
</Warning>

### Creating Custom Components

```typescript
// components/custom-card.tsx
"use client" // Only if using hooks or browser APIs

import * as React from "react"
import { cn } from "@/lib/utils"
import { cva, type VariantProps } from "class-variance-authority"

const customCardVariants = cva(
  "rounded-lg border bg-card text-card-foreground shadow-sm",
  {
    variants: {
      variant: {
        default: "border-border",
        highlighted: "border-primary"
      }
    },
    defaultVariants: {
      variant: "default"
    }
  }
)

interface CustomCardProps
  extends React.HTMLAttributes<HTMLDivElement>,
    VariantProps<typeof customCardVariants> {
  // Custom props
}

export function CustomCard({
  className,
  variant,
  ...props
}: CustomCardProps) {
  return (
    <div
      className={cn(customCardVariants({ variant }), className)}
      {...props}
    />
  )
}
```

## Performance

### Optimization Techniques

- **Code Splitting**: Lazy load routes and heavy components
- **Tree Shaking**: Vite automatically removes unused code
- **Bundle Analysis**: `npm run build -- --analyze`
- **Image Optimization**: Use Next.js Image component where possible
- **Minimize Client JS**: Default to server components (if using Next.js)

### Build Performance

- **Development**: Server starts in <1 second
- **HMR**: Updates in <100ms
- **Production Build**: 10-30 seconds (depending on app size)
- **Bundle Size**: ~300-500KB gzipped (optimized)
