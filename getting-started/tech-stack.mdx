## Overview

Zarna leverages modern, production-ready technologies across the entire stack. This page provides a comprehensive breakdown of every major technology, why it was chosen, and how it's used.

## Frontend Technologies

### Core Framework

<AccordionGroup>
  <Accordion title="React 19.1.0" icon="react">
    **Purpose**: UI library for building the user interface

    **Why React 19**:
    - Latest React with improved performance
    - Server Components support (future-ready)
    - Better TypeScript integration
    - Improved hooks performance
    - Automatic batching for state updates

    **Key Features Used**:
    - Functional components with hooks
    - Context API for global state
    - Suspense for lazy loading
    - Error boundaries for fault tolerance

    **Learn More**: [Frontend Overview](/frontend/overview)
  </Accordion>

  <Accordion title="Vite 6.2.0" icon="bolt">
    **Purpose**: Build tool and development server

    **Why Vite**:
    - Lightning-fast hot module replacement (HMR)
    - Native ES modules for development
    - Optimized production builds
    - Plugin ecosystem
    - Better DX than Create React App

    **Performance**:
    - Dev server starts in <1 second
    - HMR updates in <100ms
    - Production build in 10-30 seconds

    **Configuration**: `vite.config.ts`
  </Accordion>

  <Accordion title="TypeScript 5.x" icon="code">
    **Purpose**: Type-safe JavaScript

    **Why TypeScript**:
    - Catch bugs at compile-time
    - Better IDE autocomplete
    - Self-documenting code
    - Refactoring confidence
    - Required for modern React development

    **Configuration**: Strict mode enabled in `tsconfig.json`

    **Key Features Used**:
    - Interface definitions for props
    - Type inference
    - Generic components
    - Utility types (Partial, Pick, Omit)
  </Accordion>
</AccordionGroup>

### Styling & UI

<AccordionGroup>
  <Accordion title="TailwindCSS 4.0.9" icon="palette">
    **Purpose**: Utility-first CSS framework

    **Why Tailwind 4**:
    - Faster than previous versions
    - Better PostCSS integration
    - Improved dark mode support
    - Smaller bundle sizes
    - Excellent developer experience

    **Key Features**:
    - Utility classes for rapid development
    - Custom theme configuration
    - Dark mode with `dark:` prefix
    - Responsive design with breakpoints
    - JIT (Just-In-Time) compilation

    **Configuration**: `tailwind.config.js`
  </Accordion>

  <Accordion title="shadcn/ui (New York variant)" icon="puzzle-piece">
    **Purpose**: High-quality, accessible component library

    **Why shadcn/ui**:
    - Copy-paste components (no npm bloat)
    - Built on Radix UI primitives
    - Fully customizable
    - TypeScript-first
    - Excellent accessibility (WCAG AA+)

    **Components Used**:
    - Button, Card, Dialog, Dropdown
    - Form, Input, Select, Textarea
    - Table, Tabs, Toast, Tooltip
    - Sidebar, Navigation, Command
    - 40+ components total

    **Installation**: `npx shadcn@latest add [component]`

    **Learn More**: [Component Guide](/frontend/components)
  </Accordion>

  <Accordion title="Radix UI" icon="component">
    **Purpose**: Unstyled, accessible component primitives

    **Why Radix**:
    - WAI-ARIA compliant
    - Keyboard navigation built-in
    - Focus management
    - Screen reader support
    - Composable architecture

    **Used via shadcn/ui**: All shadcn components are built on Radix primitives

    **Key Primitives**:
    - @radix-ui/react-dialog
    - @radix-ui/react-dropdown-menu
    - @radix-ui/react-select
    - @radix-ui/react-tabs
    - 20+ primitives total
  </Accordion>

  <Accordion title="class-variance-authority (CVA)" icon="wand-magic-sparkles">
    **Purpose**: Component variant management

    **Why CVA**:
    - Type-safe variants
    - Clean API for component variants
    - Works seamlessly with Tailwind
    - Better than manual className logic

    **Example**:
    ```typescript
    const buttonVariants = cva("base-classes", {
      variants: {
        variant: { default: "...", destructive: "..." },
        size: { sm: "...", lg: "..." }
      }
    })
    ```
  </Accordion>

  <Accordion title="Lucide React 0.544.0" icon="icons">
    **Purpose**: Icon library

    **Why Lucide**:
    - 1,000+ icons
    - Consistent design
    - Tree-shakeable
    - TypeScript support
    - MIT licensed

    **Usage**:
    ```typescript
    import { Check, X, ChevronRight } from "lucide-react"
    ```
  </Accordion>

  <Accordion title="next-themes" icon="moon">
    **Purpose**: Theme management (dark/light mode)

    **Why next-themes**:
    - Zero-flash on page load
    - System preference detection
    - localStorage persistence
    - Works with SSR

    **Learn More**: [Theme System](/frontend/theme-system)
  </Accordion>
</AccordionGroup>

</AccordionGroup>

## Backend Technologies

### Core Framework

<AccordionGroup>
  <Accordion title="FastAPI" icon="bolt">
    **Purpose**: Modern Python web framework

    **Why FastAPI**:
    - Automatic OpenAPI documentation
    - Built-in validation (Pydantic)
    - Async/await support
    - High performance (comparable to Node.js/Go)
    - Type hints throughout
    - WebSocket support

    **Key Features**:
    - Dependency injection
    - Background tasks
    - Middleware support
    - CORS handling
    - Request/response models

    **Performance**:
    - Handles 10,000+ requests/sec
    - Async I/O for non-blocking operations

    **Learn More**: [Backend Overview](/backend/overview)
  </Accordion>

  <Accordion title="Uvicorn" icon="server">
    **Purpose**: ASGI server for FastAPI

    **Why Uvicorn**:
    - Async support
    - Fast and lightweight
    - Auto-reload in development
    - Production-ready with Gunicorn

    **Configuration**:
    ```python
    uvicorn.run("app.main:app",
      host="0.0.0.0",
      port=8000,
      reload=True
    )
    ```
  </Accordion>

  <Accordion title="Pydantic" icon="check-circle">
    **Purpose**: Data validation using Python type hints

    **Why Pydantic**:
    - Runtime validation
    - JSON serialization
    - Type coercion
    - Error handling
    - FastAPI integration

    **Example**:
    ```python
    class CompanyCreate(BaseModel):
        name: str
        industry: Optional[str] = None
        revenue: Optional[float] = None
    ```
  </Accordion>
</AccordionGroup>
